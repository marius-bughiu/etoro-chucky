## Tooling fix (the real work this cycle)

The import sat at **module top**, so the **parent** process hung on it *before* `main()` ran — before it could spawn the worker subprocess whose hard-kill budget is the entire safety net. The hardening guarded the worker but the parent's own top-level import silently bypassed it.
- Fix: made the yfinance import **lazy** — moved it inside `_yf_history` (the only function that touches yfinance, and it only runs in the `--worker` child). Now a wedged import hangs in the worker, where the parent's `proc.wait(timeout=30)` + `_kill_tree()` catches it.
